# AI田上 インストール手順

Slackのメンション/DMに、Copilot for Windows を使って自分の代わりに返信案を作り、
PCの作業内容をレポート化するツールです。アプリは1つの実行ファイル（`tagami.exe`）だけです。

---

## 1. 事前に用意するもの（このPCに必要）

- **Windows 11**
- **Copilot for Windows**（Microsoft Store のアプリ）… 起動してサインインしておく
- **Google Drive for Windows**（PCにドライブとして同期されていること。例: `G:\マイドライブ`）
  - 知識ベースとレポートの保存先に使います
- **Slack デスクトップアプリ**… 対象ワークスペースにログインしておく
  - ※ログイン情報はこのPC内のSlackアプリから自動取得します（クラウドAPIキー等は不要）

---

## 2. インストール

1. GitHub のリリースページを開く
   → https://github.com/VegibusIT/tagamiAi/releases/latest
2. **`tagami.exe`** をダウンロード
3. **デスクトップ**に置く（フォルダはどこでもOKだが、デスクトップ推奨）
4. `tagami.exe` を**ダブルクリック**で起動 → 「AI田上」のホーム画面が開く

> Windowsの警告（SmartScreen）が出たら「詳細情報」→「実行」で進めてください。

---

## 3. 初期設定（GUIの「⚙ 設定」タブ）

1. **返信の主体（アカウント名）**: 返信を書く本人の名前（例: `安河内`）
2. **Slackワークスペース（サブドメイン）**: SlackのURLの先頭（例: `https://vegibushq.slack.com` なら `vegibushq`）
3. **知識ベースの保存場所**: Drive上のファイルパス（例: `G:\マイドライブ\tagamiAi\knowledge.md`）
   - このファイルに「自分の役割・取引先・予定ルール・文体」を書いておくと、返信精度が上がります
   - 無ければ空ファイルでOK（後から `learn` で自動生成も可能）
4. **Slack確認の間隔**: 何秒ごとにSlackを見るか（既定180秒）
5. **「PC起動時に自動で常駐する」にチェック** → 次回ログインから自動で立ち上がります
6. **「保存」** を押す

---

## 4. 使い方（GUI）

ホーム画面の上部タブで切り替えます。

- **✉ 返信**: 最新の「自分宛メンション/DM」から返信案を自動生成 → 確認・編集して「スレッドに送信」
  - 常駐中は、新着メンション/DMが来ると自動でこの画面が開きます
- **📊 レポート**: 日付を選んで「集計」→ その日の作業内訳（アプリ別・上位の作業）
  - 「🤖 自動化候補を出す」→ Copilotが「自動化できそうな定型作業」を提案
  - 「📁 共有フォルダを開く」→ レポートの保存フォルダを開く
- **📜 ログ**: 常駐・返信・更新の動作履歴とエラーを表示（常駐は隠れて動くので、調子が悪い時はここを見る）
- **⚙ 設定**: 上記の設定 ＋「PC起動時に自動で常駐する」

※返信案の生成中、Copilotのウィンドウが画面の右下に一瞬出ます（入力に必要なため）。生成中もPCは普通に使えます。

---

## 5. レポートの共有（田上さんへ・最初の1回だけ）

レポートは自動で **`G:\マイドライブ\tagamiAi\レポート`** フォルダ（Google Drive上）に保存されます。

1. レポート画面の「📁 共有フォルダを開く」でフォルダを開く
2. そのフォルダを**右クリック →「共有」→ 共有したい相手を追加**
3. 以降、毎日のレポートは自動でこのフォルダに入るので、**相手にも自動で共有**されます

---

## 6. アップデート（自動）

**アップデートは自動です。** 常駐が起動した時と、その後6時間ごとに、GitHubの最新リリースを確認し、
新しければ自動でダウンロードして入れ替え・再起動します（オフライン時は何もせず通常動作）。

- 手動で今すぐ更新したいとき: コマンドプロンプトで `tagami.exe update`
- 現在のバージョン確認: `tagami.exe version`

---

## 7. うまく動かないとき

まず **📜 ログ** タブ（または `G:\マイドライブ\tagamiAi\logs` の当日ファイル）を見てください。常駐の開始・新着検知・送信・エラー・自動更新の履歴が時刻付きで残ります。

- **「Copilotを開けません」**: Copilot for Windows を起動してサインインしておく
- **Slackの認証エラー**: Slackデスクトップアプリでそのワークスペースにログインしておく（起動した状態で）
- **レポートが「記録が少なすぎます」**: 記録は常駐が貯めます。半日〜1日ためてから見る
- **常駐しているか確認**: タスクマネージャーに `tagami.exe` があればOK
- **自動起動を解除したい**: 設定タブのチェックを外す（または `tagami.exe autostart off`）

---

## コマンド一覧（上級者向け・通常はGUIだけでOK）

```
tagami                      ダブルクリック = GUIホーム
tagami watch                常駐（メンション監視 + 活動記録 + 自動更新）
tagami activity             活動記録だけを単体で実行
tagami report [日付] [--ai]  作業レポート（--ai でCopilotが自動化候補を提案）
tagami autostart [on|off]   ログイン時の自動起動
tagami update               今すぐ手動アップデート
tagami version              バージョン表示
```
